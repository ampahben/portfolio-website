Creating a Private Subnet in AWS

Cloud & DevOps AWS Networking

Project Overview

Enhanced AWS networking by setting up a private subnet within a Virtual Private Cloud (VPC), ensuring isolation and security with private route tables and custom network ACLs. This implementation provides a secure environment for sensitive resources while maintaining controlled access to the internet through NAT Gateway.

Challenge

Create an isolated network segment within AWS that prevents direct internet access while still allowing outbound connections for updates and patches.

Solution

Implemented a private subnet with no direct internet gateway attachment, using NAT Gateway for controlled outbound traffic and custom NACLs for subnet-level security.

Impact

Established a secure environment for databases and application servers that requires traffic to pass through security layers before reaching the internet.

Technical Implementation

Subnet Configuration

Created a private subnet (10.0.3.0/24) within an existing VPC, ensuring it doesn't have a direct route to the internet gateway. Key configuration elements:

  • Assigned CIDR block from the VPC's IP address range
  • Placed in a different Availability Zone for redundancy
  • Enabled auto-assign public IP address: false
  • Applied resource tags for identification and cost tracking

Route Table Setup

Created a custom route table for the private subnet with the following routes:

  • Local route for VPC communication (10.0.0.0/16 â†’ local)
  • Default route (0.0.0.0/0) pointing to NAT Gateway in public subnet
  • No direct route to Internet Gateway

Security Implementation

Implemented multiple layers of security:

  • Network ACLs with restrictive inbound/outbound rules
  • Security groups for instance-level traffic control
  • NAT Gateway in public subnet for controlled outbound traffic
  • VPC Flow Logs enabled for traffic monitoring
Private Subnet Architecture showing NAT Gateway integration

Results & Key Takeaways

Enhanced Security

Established a network segment where resources are inaccessible directly from the internet, reducing attack surface.

Network Segmentation

Created clear separation between public-facing and internal-only resources following AWS best practices.

Controlled Access

Implemented one-way internet access (outbound only) through NAT Gateway while blocking inbound connections.

Lessons Learned

  • Private subnets require NAT Gateway or NAT Instance for internet access
  • Route table configuration is critical for subnet isolation
  • NACLs operate at subnet level while security groups protect instances
  • Proper CIDR planning prevents IP address conflicts
  • Tagging resources systematically improves manageability
