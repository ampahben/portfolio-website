Deloitte - Cyber Investigation

Cybersecurity Incident Response

Project Overview

Investigated a data breach at Daikibo Industrials by analyzing web server logs to identify the attack vector and compromised systems, providing actionable recommendations to prevent future incidents.

Challenge

Determine the source of a sophisticated data breach affecting industrial control systems.

Solution

Conducted log analysis with advanced pattern recognition and timeline reconstruction.

Impact

Identified critical security gaps in the client's web application infrastructure.

Technical Implementation

Incident Response Phases

  1. Preparation: Established playbooks based on NIST SP 800-61r2
  2. Identification: Used Splunk correlation searches aligned with MITRE ATT&CK T1190
  3. Containment: Implemented segmentation per CIS Critical Security Control 12
  4. Eradication: Removed backdoors using Velociraptor endpoint detection
  5. Recovery: Validated systems with OpenSCAP compliance checks

Key Findings

  • Discovered API scraping through abnormal request intervals
  • Identified session hijacking through log analysis
  • Detected credential stuffing attacks against admin portals

Framework Alignment:

Applied MITRE ATT&CK Initial Access techniques (T1078, T1133) and followed ISO 27035 incident severity classification throughout the investigation.

Incident investigation workflow

Results & Key Takeaways

Vulnerabilities Found

Identified 7 critical security vulnerabilities in web application

Attackers Identified

Traced 3 distinct attacker IPs and techniques

Response Time

Reduced mean detection time from 72 to 4.5 hours

Lessons Learned

  • Web application firewalls need customized rule sets for industrial systems
  • Log retention policies must balance storage costs with investigative needs
  • Automated API scraping can be detected through timing pattern analysis
  • Incident response playbooks require regular adversary emulation testing
