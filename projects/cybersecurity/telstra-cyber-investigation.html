Telstra - Cybersecurity Management

Cybersecurity SOC Operations

Project Overview

Conducted comprehensive malware attack response as a Security Operations Centre (SOC) analyst for Telstra. Managed incident triage, threat analysis, firewall rule implementation, and post-incident documentation for a Spring4Shell vulnerability attack affecting critical infrastructure.

Challenge

Respond to active malware attack exploiting Spring4Shell vulnerability affecting critical services.

Solution

Implemented systematic incident response with threat analysis and Python-based firewall mitigation.

Impact

Successfully contained attack within 2 hours and prevented further malware spread across infrastructure.

Technical Implementation

Task 1: Incident Triage & Response

  • Analyzed firewall logs to identify affected critical infrastructure
  • Determined service priority levels for Spring Framework systems
  • Drafted incident notification emails to appropriate response teams
  • Established timeline and severity classification for the attack

Task 2: Attack Pattern Analysis

  • Investigated Spring4Shell vulnerability exploitation patterns
  • Identified distributed attack characteristics in network requests
  • Analyzed proof-of-concept payload to understand attack vectors
  • Documented technical findings for network security teams

Task 3: Technical Mitigation

  • Developed Python-based firewall rules to block malicious requests
  • Created HTTP server simulation for firewall rule testing
  • Implemented pattern-based filtering to stop malware spread
  • Validated mitigation effectiveness through controlled testing

Task 4: Incident Postmortem

  • Created comprehensive incident documentation and timeline
  • Conducted root cause analysis of the security breach
  • Documented lessons learned and improvement recommendations
  • Prepared governance and compliance audit trail
SOC incident response workflow

Results & Key Takeaways

Attack Contained

Successfully stopped malware spread within 2 hours of initial detection.

Firewall Rules Deployed

Implemented Python-based filtering rules to prevent similar future attacks.

Complete Documentation

Created detailed incident postmortem for compliance and team education.

Lessons Learned

  • Spring4Shell vulnerability requires immediate patch management prioritization
  • Distributed attacks need pattern-based rather than IP-based blocking
  • Cross-team communication is critical during active incident response
  • Automated testing of firewall rules speeds up mitigation deployment
  • Comprehensive postmortem documentation aids future incident preparation
  • SOC analysts must balance technical analysis with clear stakeholder communication
